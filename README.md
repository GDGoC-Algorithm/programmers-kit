## 💡 스터디 방식
### 문제 풀이
1. 주마다 5문제씩 선정해서 문제 풀이를 진행한 뒤, 회의 시간에 랜덤으로 1-2문제씩 풀이를 설명함. <br>
   a. 개념+알고리즘+풀이 방식 자세하게 설명하기 <br>
   b. 만약 상대가 이해 못하면 이해할 때까지 설명해야 함

### 설명 방식
1. 적용 알고리즘 개념 간단하게 설명하기
2. 문제 풀이를 위한 접근 방식(or 개념) 설명
3. 기본 코드에 대한 설명
4. 추가적으로 개선한 코드에 대한 설명
5. 사용 라이브러리 정리
6. 기타(문제 풀이에 어려웠던 점, 구현하고자 했는데 실패한 방식)

### 과제 제출
1. 깃허브 main에 개인 이름 폴더 하위에 주차별로 merge
2. 끝까지 포기하지 않고 풀어오는 것까지 과제 제출 완료.

> [!WARNING]
> *과제 제출이 이루어지지 않을 시 스터디 수료가 불가할 수 있습니다.*
> 
> 아래 항목 중 하나라도 해당할 경우 스터디 **Fail**로 간주합니다!
> 1. 스터디 기간 종료 전까지 총 7주차 워크북을 **모두 완료하지 않은 경우**
> 2. 총 8회의 온라인 미팅 중 **2회 이상** 미참석한 경우 
> (단, 사전 공유된 인정 사유가 있는 경우는 제외)
> 3. 사전 공유 없이 미팅에 불참한 경우 (**무단 결석**)
